# Processo Proposto (TO-BE)

## Venda

1. O cliente chega ao caixa.
2. O operador identifica os produtos utilizando o leitor de código de barras ou pesquisando pelo nome.
3. O sistema recupera automaticamente as informações dos produtos.
4. O sistema calcula automaticamente o valor total da venda.
5. Caso existam descontos, o operador poderá aplicá-los.
6. O cliente escolhe a forma de pagamento.
7. O pagamento é registrado no sistema.
8. O sistema registra a venda.
9. O estoque é atualizado automaticamente.
10. O comprovante da venda é impresso.
11. Os produtos são entregues ao cliente.

### Melhorias Obtidas

- Atendimento mais rápido.
- Redução de erros no registro dos produtos.
- Atualização automática do estoque.
- Registro completo das vendas realizadas.
- Histórico de vendas disponível para consulta.
- Possibilidade de calcular faturamento e lucro.
- Emissão automática de comprovantes.

---

## Compra de Mercadorias

1. A mercadoria chega ao estabelecimento.
2. O operador seleciona o fornecedor no sistema.
3. Os produtos recebidos são registrados.
4. As quantidades recebidas são informadas.
5. Os preços de compra são registrados.
6. O sistema atualiza automaticamente o estoque.
7. A compra fica registrada para consultas futuras.

### Melhorias Obtidas

- Registro completo das entradas de mercadorias.
- Atualização automática do estoque.
- Histórico de compras.
- Histórico dos preços de compra.
- Controle dos fornecedores.
- Facilidade para comparar preços entre fornecedores.

---

## Controle de Estoque

1. O estoque é atualizado automaticamente após compras e vendas.
2. O operador pode consultar a quantidade disponível de qualquer produto.
3. O sistema identifica produtos com estoque baixo.
4. O sistema registra perdas de mercadorias.
5. O sistema mantém o histórico de movimentações do estoque.

### Melhorias Obtidas

- Controle preciso das quantidades.
- Redução de perdas.
- Facilidade na reposição dos produtos.
- Histórico completo das movimentações.
- Maior confiabilidade das informações.

---

## Venda a Prazo

1. O operador seleciona o cliente cadastrado.
2. Os produtos são registrados normalmente.
3. O sistema registra a venda como "A Prazo".
4. O comprovante é impresso.
5. O cliente assina o comprovante.
6. O comprovante permanece arquivado no estabelecimento.
7. O sistema registra automaticamente o valor pendente.
8. Quando o cliente realiza o pagamento, o operador registra o pagamento no sistema.
9. O sistema atualiza automaticamente o saldo devedor do cliente.

### Melhorias Obtidas

- Controle organizado das vendas a prazo.
- Consulta rápida das dívidas dos clientes.
- Histórico completo das compras e pagamentos.
- Controle de pagamentos parciais.
- Redução do risco de perda das informações.

---

## Fechamento do Caixa

Ao final do expediente:

1. O operador registra o encerramento do caixa.
2. O sistema apresenta o resumo das vendas realizadas.
3. O sistema informa os valores recebidos em dinheiro, Pix, cartão e vendas a prazo.
4. O operador confere os valores existentes.
5. Caso exista diferença, ela poderá ser registrada.
6. O fechamento do caixa é armazenado para consultas futuras.

### Melhorias Obtidas

- Conferência mais rápida.
- Redução de erros.
- Histórico dos fechamentos.
- Facilidade para identificar divergências.
- Maior confiabilidade das informações financeiras.
